# Principles of Information Architecture

## Overview
Information Architecture (IA) is the practice of organizing, structuring, and labeling content so that users can **find**, **understand**, and **navigate** information effectively.

In documentation systems, IA determines:
- how content is grouped
- how pages relate to each other
- how users move through information
- how scalable the documentation becomes over time

---

## Core IA Principles for Documentation

### 1. Clear Content Types
Documentation should be divided into distinct types:
- Concept (what and why)
- Task (how to do something)
- Reference (syntax, parameters)
- Overview (high-level orientation)

!!! note
    Mixing content types on a single page increases cognitive load.

---

### 2. Shallow, Predictable Navigation
Users should reach most information within:
- 2–3 navigation clicks
- without guessing terminology
- without scanning long pages unnecessarily

---

### 3. One Topic per Page
Each page should answer **one primary question**.

Good examples:
- “How the CI/CD pipeline works”
- “How AI supports documentation workflows”

Poor examples:
- “Everything about documentation”

---

### 4. Progressive Disclosure
Start simple, then allow deeper exploration.

- Index pages = orientation
- Child pages = details
- Expand complexity gradually

This is why section landing pages and deep content pages exist separately.

---

### 5. Naming & Terminology Consistency
Navigation labels should:
- mirror user mental models
- avoid internal jargon
- remain stable over time

This improves both:
- usability
- search accuracy (especially AI-driven search)

---

## IA and Documentation Architecture
Information Architecture is the foundation on which:
- templates
- DocOps pipelines
- AI systems
- governance models
are built.

Poor IA limits automation and AI effectiveness.

---

## Summary
Good Information Architecture:
- reduces friction
- scales across teams
- enables automation
- improves discoverability
- supports AI-assisted documentation

In modern documentation systems, IA is not optional — it is structural.
